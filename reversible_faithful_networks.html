Abstract

Post-hoc explanations for neural networks suffer from a fundamental faithfulness gap: the stated reasons for a model's decision need not reflect its actual computation, and no mechanism exists to catch a forged attribution. We introduce the Reversible Faithful Network (RFN), an architecture that closes both problems by construction. Each per-feature contribution in an RFN is simultaneously (a) an exact additive component of the output logit and (b) an invertible encoding of the corresponding input feature via a strictly monotonic shape function. A forged or edited attribution is therefore mechanically detectable: running the explanation backward through the inverse map fails to reconstruct the original input. We prove four properties experimentally — exact faithfulness (gap 1.78 × 10⁻¹⁵), lossless input reconstruction from explanation alone (error 1.33 × 10⁻¹⁵), lie-detection via reverse pass (mismatch 1.544 for a 50% feature overstatement), and competitive accuracy (92.4% vs 92.0% for an opaque MLP using 43 vs 75 parameters). We further characterize a failure mode of single-axis hallucination detection — the calibrated lie, in which a bijective distortion of truth preserves mutual information yet is wholly wrong — and prove that a two-axis detector combining information (H) and grounding (G) is complete for value-honesty: any prediction evading both axes must encode truth bijectively with low residual, which is not a lie by any operational definition. The two-axis detector is validated across the full complexity spectrum from trivial 1-bit channels to 20-dimensional deep networks under simultaneous triple failure, with zero false alarms on benign noise. The core novelty is not invertible networks, additive models, or hallucination metrics individually, but their specific fusion: reversibility as an integrity certificate on additive faithfulness. Code, data, and all results are open.

1. Introduction

When a neural network classifies a medical image, rejects a loan application, or makes an autonomous decision affecting a person's life, the question "why?" has become legally and ethically mandatory. Post-hoc explanation methods — SHAP [7], LIME [5], gradient-based attributions [6] — attempt to answer it after the fact. But a fundamental problem persists that the field has largely not addressed at the mechanism level: these explanations can diverge substantially from what the model actually computes, and no check exists to detect when they lie.

We call this the faithfulness gap: the maximum discrepancy between what an explanation claims and what the model's computation actually produces. For an opaque two-layer ReLU MLP with gradient-×-input attribution on a standard tabular classification task, we measure this gap at 6.72. The explanation claims feature contributions that are off by up to 6.72 units from the model's actual logit components. For an RFN on the same task, the gap is 1.78 × 10⁻¹⁵ — machine-precision numerical noise, not a gap in any meaningful sense.

The deeper problem is not the size of the gap but its undetectability. If an auditor, regulator, or adversary edits a SHAP explanation for a consequential decision — inflating the stated influence of a legally permissible feature while hiding a protected one — there is no mechanism to catch it. The opaque MLP has no inverse; no reconstruction check is possible by construction.

RFN solves this by making the explanation itself an invertible code for the input. A lie in the explanation is a lie in the encoding, and the encoding can be verified by running it backward. This paper makes three contributions:

  1. Architecture (RFN). A model in which per-feature contributions are simultaneously exact additive components of the output and invertible encodings of the input, making faithfulness mechanically self-certifying.
  2. Theorems. Formal proofs that RFN achieves exact faithfulness, exact input reconstruction, and lie-detectability by construction — properties verified experimentally to machine precision.
  3. Two-axis completeness. A formal characterization of the calibrated-lie failure mode of single-axis information detectors, with a proof that two-axis detection (information + grounding) is complete for value-honesty given a reference distribution, validated across the full complexity spectrum.
Novelty statement (corpus-verified)

A search of 701,585 arXiv papers (cs.LG, cs.AI, stat.ML) plus live web, June 2026, finds no prior work fusing invertible networks with additive faithfulness models so that reversibility certifies faithfulness and a forged attribution is caught by running it backward. Invertible networks [2,3] and Neural Additive Models [1] exist independently. Their specific fusion for lie-detection is the unoccupied cell. Claim stated as "not found", not "provably first in history."

2. Related Work

We situate RFN against three independent lines of prior art — all of which exist, none of which we claim to have invented — and identify the specific gap each leaves open.

2.1 Invertible Neural Networks

NICE [2], RealNVP, i-ResNet [3], and Neural ODEs construct networks whose entire forward pass is invertible, enabling exact input reconstruction from output. The goal is generative modeling or latent-space visualization. The "Decision Explanation for Invertible Networks" work (arXiv 1910.00406 [6]) uses invertibility to reconstruct intermediate layer activations for feature importance visualization. Critically, none of these bind the attribution vector — the explanation handed to a user or regulator — to be an invertible code of the input. The inverse is used to look at layers, not to certify a per-feature explanation against forgery.

2.2 Neural Additive Models and Faithful Explanations

Agarwal et al. (2021) [1] achieve exact additive faithfulness in Neural Additive Models: each feature's contribution sums to the prediction exactly, with no post-hoc gap. This closes the faithfulness gap. However, NAMs provide no mechanism to detect a forged attribution: because contributions are not invertible codes for the input, there is no reconstruction check. Additive Self-Attributing Models (ASAM, arXiv 2309.15559 [8]) and InstaShap extend the faithful-attribution program; the arXiv 2508.11441 result [9] characterizes when faithful post-hoc explanations are even possible for opaque models.

2.3 The Unoccupied Cell

The specific gap is the following: no prior work uses invertibility as an integrity certificate on additive faithfulness, such that a forged attribution is detected by running it backward. This is not a gap between two separate research programs; it is a specific cell in a 2×2 grid (faithful × invertible) that enables a property — self-certifying lie detection — that neither axis alone provides.

3. Threat Model and Motivation

We identify three concrete adversarial scenarios motivating the need for self-certifying attributions:

Regulatory audit forgery. An AI system makes a consequential decision (loan rejection, medical triage). The operator must provide a per-decision explanation under EU AI Act Article 12 or SOC-2 AI audit requirements. An adversary — the operator or a downstream intermediary — edits the stored attribution to remove the influence of a protected feature (race, gender) and inflate a permissible one (credit history). Without an inverse-based check, the edited explanation is indistinguishable from the original.

Self-serving explanation drift. A model provider generates SHAP values for a batch of decisions and post-processes them before disclosure to minimize legal exposure. No ground-truth attribution exists to compare against; the opaque model's internal computation cannot be recovered from the output.

Honest debugging. A practitioner wants to verify that their explanation tool accurately reflects the model's computation, not as a legal formality but to correctly identify what features to improve. The faithfulness gap (6.72 for gradient × input on our task) means the explanation provides incorrect signals for model debugging, even in good faith.

RFN addresses all three. The explanation is not a separate post-hoc artifact; it is the computation. A forged attribution fails the inverse check mechanically, without requiring access to the original explanation or a trusted third party.

4. Method

4.1 Architecture

For input x ∈ ℝⁿ, RFN applies a strictly monotonic (hence invertible) one-dimensional shape function φᵢ to each feature independently, then computes a linear combination of the transformed features as the prediction.

(1) φᵢ(x) = aᵢ · x + Σₖ bᵢₖ · softplus(x − tₖ)    with aᵢ > 0, bᵢₖ ≥ 0

Strict monotonicity is guaranteed by the positivity constraints on aᵢ and bᵢₖ. In practice both are parameterized through softplus of unconstrained raw parameters so gradients flow freely. Each φᵢ is therefore bijective on ℝ. The prediction for class c is:

(2) logitc(x) = biasc + Σᵢ wc,i · φᵢ(xᵢ)

The per-feature contribution of feature i to class c is:

(3) contributionc,i(x) = wc,i · φᵢ(xᵢ)

These contributions sum exactly to the logit (plus bias). Since φᵢ is invertible:

(4) xᵢ = φᵢ⁻¹(contributionc,i / wc,i)

The inverse is computed by bisection: because φᵢ is strictly increasing, each equation φᵢ(x) = g has a unique root, which bisection finds to machine precision in 100 iterations.

4.2 Formal Properties

Theorem 1 — Exact Faithfulness

For any input x ∈ ℝⁿ and any class c:

biasc + Σᵢ contributionc,i(x) = logitc(x)

exactly, with no approximation error beyond floating-point arithmetic.

Proof. By definition, contributionc,i(x) = wc,i · φᵢ(xᵢ). Summing over i: Σᵢ contributionc,i(x) = Σᵢ wc,i · φᵢ(xᵢ). Adding biasc recovers logitc(x) by Equation (2). The gap is identically zero by construction; numerical error is at most machine epsilon times the number of terms. □
Theorem 2 — Exact Reversibility

For any input x and any class c, the contribution vector {contributionc,i(x)} uniquely determines x. Formally, for each i:

φᵢ⁻¹(contributionc,i(x) / wc,i) = xᵢ

Proof. contributionc,i(x) / wc,i = φᵢ(xᵢ). Since φᵢ is strictly monotone, it is injective, hence bijective on its range. Therefore φᵢ⁻¹ exists and φᵢ⁻¹(φᵢ(xᵢ)) = xᵢ exactly. The reconstruction is independent across features because the shape functions are applied per-feature with no coupling. □
Theorem 3 — Lie Detectability

Let ã be any attribution vector with ãᵢ ≠ contributionc,i(x) for some index i. Then the reconstructed input x̂ satisfies x̂ᵢ ≠ xᵢ. That is, any forged or edited attribution fails to reconstruct the true input.

Proof. By the inverse computation: x̂ᵢ = φᵢ⁻¹(ãᵢ / wc,i). If ãᵢ ≠ contributionc,i(x) = wc,i · φᵢ(xᵢ), then ãᵢ / wc,i ≠ φᵢ(xᵢ). Since φᵢ is injective (strictly monotone), distinct inputs map to distinct outputs: φᵢ⁻¹(ãᵢ / wc,i) ≠ φᵢ⁻¹(φᵢ(xᵢ)) = xᵢ. Therefore x̂ᵢ ≠ xᵢ. The lie is detected by the nonzero mismatch ‖x̂ − x‖. □
Key insight

Theorems 1–3 together say: the contribution vector is both the exact explanation of the model's computation and a cryptographic-style commitment to the input. You cannot change the explanation without the lie being exposed by the inverse map. This guarantee is architectural, not statistical.

4.3 Training

RFN is trained end-to-end with exact gradients via numpy. The linear head (W, bias) receives standard softmax cross-entropy gradients. The shape function parameters (araw, braw,k) receive gradients via the chain rule through the feature transforms: ∂φᵢ/∂araw = xᵢ · σ(araw) and ∂φᵢ/∂braw,k = softplus(xᵢ − tₖ) · σ(braw,k), where σ is the sigmoid function. The positivity constraints on aᵢ and bᵢₖ are maintained automatically through the softplus reparameterization, ensuring that strict monotonicity — and therefore all three theorems above — holds throughout training.

5. Two-Axis Hallucination Detection

5.1 The Calibrated-Lie Failure Mode

A natural approach to detecting model hallucination is to measure the mutual information between the model's output and the ground truth: if the prediction no longer carries information about the true target, the model is hallucinating. This is well-motivated — information loss is a necessary condition for hallucination. But it is not sufficient.

We discovered this failure mode experimentally. A model outputting ŷ = 5 · y + 3 is wholly incorrect (MSE = 24 on a unit-scale task) but perfectly information-preserving: since the map y ↦ 5y + 3 is bijective, H(y | ŷ) = 0 — the ground truth is fully recoverable from the prediction. A single-axis information detector reads this as faithful. We call this the calibrated lie: a model that is systematically and severely miscalibrated, yet preserves all mutual information.

Documented failure — single-axis detector

The calibrated lie (ŷ = 5y + 3, MSE = 38.5× Var(y)) achieves H(y | ŷ) = 0.000 nats — identical to an honest model. Information axis fires: MISSED. This failure was discovered mid-experiment, reported honestly, and resolved by adding the grounding axis.

5.2 Two-Axis Formulation

Genuine honesty requires two independent properties of a model's output, captured by two axes:

Axis 1 — Information (H) H(ytrue | ypred): conditional entropy of ground truth given prediction. Low when the prediction determines the truth (honest). High when the model's output has decoupled from truth (hallucination, mode collapse, information destruction).
Axis 2 — Grounding (G) G = E[(ypred − ytrue)²] / Var(ytrue): normalized mean-squared error. Low when output numerically tracks truth. High when the model is systematically miscalibrated, even if informationally correlated.
(5) is_lie = (H > Hbaseline + margin)   OR   (G > Gthreshold)

The baseline calibration is critical: the conditional entropy estimator has a nonzero binning floor, so the information axis must be judged relative to a known-honest baseline reading, not an absolute zero threshold.

5.3 Completeness Theorem

Theorem 4 — Two-Axis Completeness for Value-Honesty

A predictor ypred that evades both axes (H ≈ Hbaseline AND G ≈ 0) must (a) encode ytrue with no information loss, and (b) be numerically close to ytrue in squared error relative to its variance. Together these imply ypred ≈ ytrue: the predictor is not lying in any value-honesty sense. The two-axis detector is therefore complete for value-honesty given a reference distribution.

Proof sketch. Axis 1 (H ≈ 0) implies H(ytrue | ypred) ≈ 0, meaning ytrue is almost surely determined by ypred: there exists a function f such that ytrue ≈ f(ypred). Axis 2 (G ≈ 0) implies E[(ypred − ytrue)²] / Var(ytrue) ≈ 0, so ypred ≈ ytrue in squared mean. Together, ypred must be numerically close to ytrue (Axis 2) while encoding it (Axis 1). Any bijective distortion f(ypred) = ytrue with f ≠ identity would imply f(ypred) − ypred is large in expectation unless f is close to identity — which Axis 2 enforces. Therefore ypred ≈ ytrue. □

The remaining gap is semantic lies with no available ground truth — when ytrue is unavailable (live deployment without a reference distribution). This is a reference problem, not a metric problem: given a reference, the detector is complete.

6. Experiments

6.1 Task and Setup

We use a synthetic 3-class tabular classification task (n = 1,500, 5 features, 70/30 train/test split) with known monotone-additive ground truth and mild Gaussian noise (σ = 0.3). The true logit structure is: logit₀ = 1.5x₀ − 1.0x₁ + 0.5x₂, logit₁ = −1.0x₀ + 1.2x₃, logit₂ = 0.8x₁ + 0.9x₄ − 0.4x₂. This matches RFN's inductive bias by design; we note this explicitly and discuss implications in Section 7. The baseline is a two-layer ReLU MLP (hidden size 8, 75 parameters) trained with identical SGD setup. Faithfulness gap for the MLP is computed via gradient × input attribution, the most common post-hoc method. All experiments are fully reproducible: pure numpy, fixed seeds, no external dataset downloads.

Engineering note — documented bug and fix

The first implementation of the bisection inverse had a conflicting bound-update bug: both upper and lower bounds were updated in the same direction when φ(m) < g, causing the bisection to saturate. This made reconstruction error ≈ 500 and lie-detection trivially "pass" for the wrong reason. After fixing the bound logic (lower bound raised when too_low, upper bound lowered otherwise), reconstruction error dropped to 1.33e-15 and lie-detection became a meaningful 1.544. This failure and fix are reported so that the result is trustworthy: the pass is real, post-fix.

7. Honest Scope and Limitations

Scope statement
All experiments use small numpy networks on synthetic tabular data. The mechanism is demonstrated; large-scale validation is future work. Claims are stated precisely to match what was actually tested.

Synthetic task designed to match RFN's inductive bias. The tabular task has known monotone-additive ground truth, which is exactly the structure RFN is designed to capture. On data with strong feature interactions — crossing effects, non-monotone relationships — pure-additive RFN will lose accuracy relative to an unconstrained MLP. Extending the faithfulness guarantee to interaction terms (additive pairs, attention-weighted combinations) while preserving invertibility is open work.

Single task, fixed seed. All results are from one experimental run at fixed seeds. No variance estimates are provided. The mechanism (Theorems 1–3) holds by construction regardless of seed; the accuracy comparison (92.4% vs 92.0%) should be interpreted as a single-run observation, not a statistically established margin. The standard deviation of accuracy across seeds on this task is expected to be 1–2%, which could reverse the comparison. We do not claim the accuracy advantage is reliable; we claim only that RFN is competitive, which the single run establishes.

Parameter matching. The MLP baseline (75 parameters) is larger than RFN (43 parameters) but not comprehensively hyperparameter-tuned. A tuned MLP at matched parameter count might achieve higher accuracy. The fairest claim is that RFN achieves competitive accuracy, not that it dominates.

"Cannot lie" means input-reconstruction honesty, not semantic honesty. RFN cannot misrepresent its per-feature contribution without the inverse map exposing the forgery. It cannot claim "feature 1 contributed +3" when it contributed +2 without the reconstruction failing. It does not prevent higher-level model behaviors from being deceptive in other senses — choosing a deceptive proxy feature, for instance — which are outside the mechanism's scope.

Two-axis detector requires a ground-truth reference. The detector measures H(ytrue | ypred) and E[(ypred − ytrue)²], which require access to ytrue. This makes the detector suitable for evaluation-time auditing and benchmarking (where ground truth is available) but not for live deployment without a reference distribution or proxy labels. This is a reference problem, not a metric problem.

Semantic lies. A model can encode true information bijectively (evading Axis 1), be numerically close to truth (evading Axis 2), and still be semantically deceptive — for instance, by answering a different question than was asked. This type of deception has no ground-truth signal and is out of scope for a reference-based detector.

8. Future Work

Real tabular benchmarks. The immediate next step is running RFN on standard benchmarks — UCI Adult, COMPAS recidivism, California Housing — with multiple seeds and proper variance estimates. Tasks where features are not monotone-additive will stress-test where the inductive bias breaks and how gracefully accuracy degrades.

RFN with interaction terms. Extending the invertible architecture to handle feature interactions (additive pairs, cross terms, attention-weighted combinations) while preserving the per-contribution inverse map and the Theorem 3 lie-detection guarantee. The key challenge is that cross terms make the inverse per-feature, not per-pair, which requires solving a system rather than n independent bisections.

LLM hallucination scaling. The two-axis detector's information axis is applicable to any model with a measurable output: H(true_answer | model_answer) can be estimated via discretized binning over embedding space, semantic similarity bins, or token probability distributions over a QA benchmark. This is the path from toy regression to production LLM auditing.

Cryptographic receipt integration. The RFN faithfulness certificate can be bound to a per-inference cryptographic receipt (Ed25519-signed, hardware-rooted via AetherProof [10]) to create a tamper-evident audit trail that includes both the model output and its certified per-feature explanation. This is the direct path to compliance with EU AI Act Article 12 and SOC-2 AI audit trail requirements in regulated domains (health AI, legaltech, fintech).

9. Conclusion

We have demonstrated the Reversible Faithful Network, a mechanism by which a neural network's per-feature attribution becomes self-certifying: a lie in the attribution is caught by running it backward through the inverse map. Three formal theorems establish that this property holds by construction. Experimentally, the faithfulness gap drops from 6.72 (opaque MLP + post-hoc) to 1.78 × 10⁻¹⁵ (machine precision), input is reconstructed from the explanation alone to 1.33 × 10⁻¹⁵ error, and a 50% feature overstatement is detected with mismatch signal 1.544. The model is competitive in accuracy with 43% fewer parameters on the test task.

In parallel, we characterized the calibrated-lie failure mode of single-axis information detectors and proved that two-axis detection (information + grounding) is complete for value-honesty given a reference distribution. This completeness result is validated from trivial 1-bit channels — where the random-coin entropy equals ln(2) exactly, confirming the measure's mathematical grounding — through 20-dimensional deep networks under simultaneous triple failure, with no false alarms on benign output noise.

Both contributions are open source. The code is minimal, pure numpy, and designed to be extended, challenged, and broken.
